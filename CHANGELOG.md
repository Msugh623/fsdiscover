# Changelog

## 2026-06-01 — v_1.9.0 

Summary:

- Final UI and runtime updates for v1.9.0. This release consolidates an
  intermediate UI-fix pass and the final updates that introduce Safe Mode
  support (frontend landing + upload flow), runtime configuration additions,
  admin UI improvements, and various asset updates.

Highlights:

- Safe Mode: added a Safe Mode upload-only landing (`fe/src/pages/SafeMode.jsx`)
  and a frontend guard in `fe/src/Layout.jsx` that redirects unauthenticated
  users to `/login` when Safe Mode is enabled.
- Upload compatibility: Safe Mode uploader aligned with FileManager (`/fs/upload`,
  `files` form field) and a new `safemodeUploadDir` runtime field exposed in the
  admin UI and init flow.
- UI polish & restructuring: FileManager components (Sidebar, Header, Main
  Section, DirItem/File/Folder), touchpad components, and admin pages received
  styling and structural improvements; several unused components were removed
  and CSS consolidated.
- Bugfix: fixed parsing/JSX issues in `fe/src/components/Loader.jsx`.
- Backend/runtime: `utils/handlers.js`, `utils/schemas.js`, and
  `utils/useRuntimeConfig.js` updated to persist new runtime fields.

Files changed (combined summary):
`fe/src/pages/SysAdminIndex.jsx`, `fe/src/pages/Init.jsx` updated to expose and
persist `safemodeUploadDir` and other runtime fields.

- Runtime/backend changes: `utils/handlers.js`, `utils/schemas.js`, and
  `utils/useRuntimeConfig.js` updated to include and persist `safemodeUploadDir`.
- Public client assets updated (several `public/client/assets/*` changes).
